# Right-Panel Grouped Neuropil Summary Design

**Goal:** Define a strict `right-panel grouped neuropil summary（右侧面板分组神经纤维区摘要）` design so the experiment console’s right-side brain panel matches the active `3D neuropil glow（3D 神经纤维区发光）` semantics while preserving access to `formal fine-grained neuropils（正式细粒度神经纤维区）`.

**Scope:** This design covers the `experiment console（实验控制台）` right-side brain detail panel only. It does not change the backend truth chain, the 3D asset contract, or the grouped 3D glow rendering path.

## 1. Problem Statement

The repository now has a correct V1 `3D neuropil glow（3D 神经纤维区发光）` path:

- the 3D viewport consumes backend-declared `display_region_activity`
- the grouped 3D asset manifest contains the approved eight grouped neuropils
- grouped glow follows the approved `FlyWire official route（FlyWire 官方路线）`

The current right-side brain detail panel still mixes two different semantic layers:

- the 3D viewport renders the grouped V1 display layer from `brainView.display_region_activity`
- the right panel’s “top regions” list renders `brainView.top_regions`, which is derived from fine-grained `region_activity`

This creates a user-facing semantic mismatch:

- the 3D viewport lights up `AL / LH / PB / FB / EB / NO / LAL / GNG`
- the right panel shows entries such as `ME_R / LO_L / LOP_R`
- users reasonably assume these are the same layer, but they are not

This is not a truth-chain bug. It is a presentation bug caused by using a fine-grained formal ranking as the default right-panel summary.

For this project, the right panel must not blur the boundary between:

- `formal truth（正式真值）`
- `display transform（显示变换）`
- `3D rendering（3D 渲染）`

---

## 2. Current State

The current UI wiring is:

- `BrainShellViewport` consumes `brainView.display_region_activity`
- the right-panel “top regions” list consumes `brainView.top_regions`
- `top_regions` is generated by sorting fine-grained `region_activity`, not grouped display payloads

This means the same card currently presents:

- grouped 3D semantics on the left
- fine-grained formal ranking on the right

The current “区域清单” row also uses `brainAssets.neuropil_manifest.length`, which is the grouped 3D mesh manifest count. It is therefore a grouped display value, not a count of formal fine-grained regions.

---

## 3. Approved Design Principles

The approved right-panel fix follows these principles:

- the default right-panel summary must align with the active 3D glow layer
- fine-grained formal detail must remain available for research inspection
- the right panel must make the distinction between grouped display semantics and fine-grained formal semantics explicit
- no new truth-layer inference may be added in the frontend
- the grouped summary must consume backend-declared `display_region_activity`, not regroup fine-grained entries on the client
- the UI must remain in `research strict mode（科研严格模式）`

---

## 4. Approach Comparison

### Approach A: Formal-first

Keep the current fine-grained ranking as the default summary and add a small grouped note or grouped badge.

Pros:

- minimal UI change
- preserves current formal detail visibility

Cons:

- keeps the existing user confusion
- does not align the main right-panel summary with 3D glow

### Approach B: Grouped-first with collapsible formal detail

Make grouped V1 activity the default right-panel summary and move fine-grained formal ranking into a collapsed secondary section.

Pros:

- aligns the main right-panel view with the 3D layer
- preserves formal fine-grained detail
- makes semantic layering explicit

Cons:

- adds one more structural section to the right panel

### Approach C: Equal-weight tabbed split

Add tabs or toggles for “Grouped” vs “Formal”.

Pros:

- clean separation of modes

Cons:

- hides important context behind extra interaction
- weakens the default semantic signal

### Recommended approach

Recommend **Approach B**.

This gives the right panel a clear default semantic layer that matches the 3D glow while preserving fine-grained scientific detail as an explicitly secondary layer.

---

## 5. Information Architecture

The right-side brain detail panel should become a two-layer structure.

### 5.1 Primary layer: Grouped Neuropil Summary

This section is always expanded and is the default semantic layer for the card.

It should include:

- `Afferent / Intrinsic / Efferent activity（输入层 / 中间层 / 输出层活动）`
- `Top grouped neuropils（最活跃分组神经纤维区）`
- `Coverage（覆盖度）`
- `Provenance（来源追踪）`
- `Shell asset / shell format / grouped manifest count（脑壳资产 / 格式 / 分组清单数量）`

The grouped region list must come from sorted `display_region_activity`, not from `top_regions`.

This section answers:

`What is the active 3D glow layer showing right now?`

### 5.2 Secondary layer: Formal Neuropil Detail

This section is collapsed by default.

It should include:

- `Top formal neuropils（最活跃正式细粒度神经纤维区）`
- existing fine-grained `top_regions` activity summaries
- a short explanatory sentence stating that this is formal fine-grained detail and not the grouped 3D layer

This section answers:

`What does the fine-grained formal neuropil ranking look like underneath the grouped display transform?`

## 6. Field Contract

### 6.1 Primary grouped summary fields

The default summary must consume only:

- `brainView.display_region_activity`
- `brainView.mapping_coverage`
- `brainView.formal_truth`
- `brainView.artifact_contract_version`
- `brainView.artifact_origin`
- `brainAssets.neuropil_manifest.length`

The grouped list must not consume:

- `brainView.top_regions`
- `brainView.region_activity`

### 6.2 Secondary formal detail fields

The collapsed formal section may consume:

- `brainView.top_regions`
- optionally `brainView.region_activity` in a future extension

This section is allowed to display fine-grained region ids such as `ME_R` and `LOP_R`, because its job is formal scientific inspection rather than 3D alignment.

---

## 7. Labeling and Copy

### 7.1 Section titles

Change the current generic explanation framing into more explicit titles:

- primary title: `Neuropil Activity Summary（神经纤维区活动摘要）`
- secondary title: `Formal Neuropil Detail（正式神经纤维区明细）`

### 7.2 Grouped summary copy

Add a short line beneath the primary title:

- `与 3D 发光层一致的 8 区分组活动摘要`

### 7.3 Formal detail copy

Add a short note inside the collapsible section:

- `用于科研细查；与 3D 的 8 区分组显示不同。`

### 7.4 Manifest label

Rename the current `区域清单` metric to something that explicitly matches the grouped 3D asset layer:

- `3D 分组清单`

This prevents users from misreading it as a count of fine-grained formal neuropils.

---

## 8. Interaction Model

The interaction model should remain intentionally light.

- the grouped summary is always visible
- the formal detail is collapsed by default
- the formal detail may be expanded on demand with a lightweight `Accordion（折叠面板）` or `Collapsible（可展开区块）`
- no tabs are needed
- no hover or click coupling between fine-grained formal rows and grouped 3D glow is needed in V1.1

This avoids introducing new interactions that imply a one-to-one mapping between fine-grained formal entries and grouped 3D meshes.

---

## 9. Failure Semantics

The right panel must remain fail-closed.

- if `display_region_activity` is unavailable, the grouped summary section should show unavailable state rather than infer grouped activity from fine-grained regions
- if the grouped 3D layer has fallen back to shell-only, the grouped summary may still show formal grouped payloads if they are present, but the wording should remain consistent with the 3D viewport state
- the formal fine-grained detail section must never be promoted as if it were the grouped 3D display layer

---

## 10. Testing Strategy

The implementation should be verified with:

- a component test that confirms the grouped summary renders grouped ids such as `AL` and `FB` rather than fine-grained ids
- a component test that confirms the formal detail section is collapsed by default
- a component test that confirms expanding the formal detail reveals fine-grained `top_regions`
- a component test that confirms the grouped manifest metric label reflects grouped 3D semantics
- a real browser smoke test that confirms the right-panel grouped summary matches the active 3D grouped glow interpretation

---

## 11. Out of Scope

This design does not include:

- changing backend truth semantics
- changing `top_regions` generation on the backend
- adding left/right laterality-specific 3D meshes
- adding hover-linked cross-highlighting between 3D and the formal detail section
- reworking the overall card layout beyond the right-panel information structure

---

## 12. Final Design Summary

The approved V1.1 right-panel fix is:

- default right-panel summary becomes grouped-first and matches the active 3D glow semantics
- the grouped summary consumes backend-declared `display_region_activity`
- the fine-grained formal ranking remains available in a collapsed `Formal Neuropil Detail` section
- grouped display semantics and formal fine-grained semantics are made visually and textually explicit

This keeps the UI honest to the scientific data chain while removing the current user-facing ambiguity.
